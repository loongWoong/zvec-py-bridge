# 知识图谱能力增强计划

## 目标
1. **后端图查询算法增强**：利用 SurrealQL 原生多跳图遍历 + Python 层图算法，提供更多样的知识图谱查询能力
2. **前端展示效果提升**：用 D3.js 替换纯 SVG 圆形布局，实现力导向/层次/环形聚焦/类型聚类四种布局切换 + 缩放/拖拽/平移/高亮/边方向/tooltip

## 一、后端图查询算法增强（wiki_runtime.py + app.py）

### 1.1 修复现有图查询缺陷
- **`neighbors(depth=)` 死参数修复**：用 SurrealQL 多跳语法 `->|edge_type|depth|->` 实现真正的深度遍历（若语法不支持则降级为 Python BFS）
- **`graph_subtree` 边去重**：用 `frozenset({from,to,edge_type})` 去重，避免双向访问重复
- **`graph_subtree` 性能优化**：用单次多跳查询替代递归 24×N 条 SQL

### 1.2 新增图查询算法（对应 design.md 图结构设计）

| 函数 | 算法 | 说明 |
|------|------|------|
| `shortest_path(src, dst)` | BFS 最短路径 | 两文档间的最短关系路径（支持跨 entity 节点），返回路径节点序列 |
| `common_neighbors(node_a, node_b)` | 共同邻居 | 两节点的共同邻居（共同引用的实体、共同标签等），发现隐含关联 |
| `node_degree(node_id)` | 度中心性 | 节点的入度/出度/总度，衡量节点重要性 |
| `top_central_nodes(limit)` | 度中心性排序 | 知识库中最重要的节点（度最高的 hub 节点） |
| `graph_stats()` | 图统计 | 节点数/边数/各边类型计数/平均度/密度，全局图概览 |
| `entity_co_occurrence(entity_name)` | 共现分析 | 与某实体共同出现在文档中的其他实体（通过 mentions 反查） |
| `knowledge_lineage(doc_id)` | 知识血缘 | 文档的上下游知识链（递归遍历 extends/depends/related，对应 design.md "展示Embedding上下游知识"） |
| `multi_hop_neighbors(node_id, depth)` | 多跳邻接 | 真正的 N 跳邻接（BFS），返回按层级分组的节点 |

### 1.3 新增 API 路由（app.py）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/api/graph/{id}/shortest-path?target={id}` | 最短路径 |
| GET | `/api/graph/{id}/common?other={id}` | 共同邻居 |
| GET | `/api/graph/{id}/degree` | 节点度 |
| GET | `/api/graph/central?limit=10` | 中心节点 Top-N |
| GET | `/api/graph/stats` | 全局图统计 |
| GET | `/api/entities/{name}/co-occurrence` | 实体共现 |
| GET | `/api/documents/{id}/lineage` | 知识血缘链 |
| GET | `/api/graph/{id}/multi-hop?depth=3` | 多跳邻接（按层级） |
| GET | `/api/graph/full` | 全图数据（供前端 D3 渲染，含所有节点+边） |

### 1.4 图检索路增强（`_graph_search`）
当前只做"实体名子串匹配→mentions 反查"。增强为：
- ① 实体名匹配 → mentions 反查文档（现有）
- ② 命中文档后，沿 related/extends/depends 边扩展 1 跳，召回关联文档（图扩展检索）
- ③ 实体间 entity_related 边反查关联实体再 mentions 反查

## 二、前端展示效果提升（static/index.html）

### 2.1 引入 D3.js（CDN）
```html
<script src="https://d3js.org/d3.v7.min.js"></script>
```

### 2.2 四种布局切换（布局选择器 UI）
1. **力导向布局（Force）**：D3 forceSimulation，spring + charge + center 力，节点自然分散。默认布局。
2. **层次布局（Hierarchy）**：按关系边层级分层（D3 tree/dagre 风格），从根节点向下展开，适合查看依赖链。
3. **环形聚焦布局（Radial）**：选中某节点为中心，按 BFS 距离分层向外辐射（d3.radialPlacement）。
4. **类型聚类布局（Cluster）**：按节点类型（document/entity/tag/topic）分 4 个扇区聚类，同类型聚在一起。

布局选择器：顶部 toolbar 下拉/按钮组，切换时平滑过渡（D3 transition）。

### 2.3 交互增强
- **缩放/平移**：D3 zoom（滚轮缩放 + 拖拽平移画布）
- **节点拖拽**：拖拽节点固定位置（force 布局下拖拽时固定）
- **Hover 高亮**：hover 节点时高亮其邻接边和邻居节点，其余变灰
- **Tooltip**：hover 显示节点详情（类型、度数、摘要）
- **边方向箭头**：SVG marker 定义箭头，显示关系方向
- **边类型标签**：边上显示关系类型（related/extends/depends 等），可开关
- **节点大小映射度数**：度越高的节点越大（hub 节点视觉突出）
- **双击聚焦**：双击节点切换为环形聚焦布局，以该节点为中心
- **节点类型过滤**：保留现有 4 个 checkbox，增加边类型过滤

### 2.4 全图数据加载
新增 `/api/graph/full` 端点，一次性返回所有节点（document/entity/tag/topic）和所有边，供 D3 渲染完整知识图谱。替代当前从 `/api/documents` + `/api/entities` 拼凑的方式。

### 2.5 清理死代码
删除 buildGraph() 中第 521-524 行被覆盖的死代码，重构为 D3 数据驱动模式。

## 三、实现顺序

1. **wiki_runtime.py** — 修复 neighbors depth + graph_subtree 去重/性能；新增 8 个图查询算法；增强 _graph_search；新增 graph_full()
2. **app.py** — 新增 9 个图查询 API 路由
3. **static/index.html** — 引入 D3.js CDN；重写知识图谱视图（D3 force + 4 种布局切换 + 交互增强）；新增图统计面板
4. **验证** — 启动服务，测试所有新 API 和前端布局切换

## 四、验证标准
- `/api/graph/full` 返回全图节点+边
- `/api/graph/{id}/shortest-path` 返回路径
- `/api/graph/stats` 返回图统计
- 前端 4 种布局可切换，缩放/拖拽/平移/高亮/tooltip 正常
- 图检索路召回提升（命中文档的关联文档也被召回）

## 五、备注
- SurrealQL 多跳语法（`->|edge|N|->`）若在嵌入式模式不支持，图算法降级为 Python BFS/DFS（基于单跳 neighbors 拼装），不影响功能正确性，仅影响性能。
- D3.js 通过 CDN 引入，无需 npm/打包，延续项目零前端构建风格。
- 图算法在 Python 层实现（BFS/DFS/度中心性/共同邻居），因为 SurrealDB 嵌入式模式可能不支持复杂图查询函数，Python 层实现更可控。
