## 三项需求实现计划

### 需求 1：持久化存储 — 重启不需要重新初始化

**`agent.py`**：
- 新增 `STATE_FILE` 路径常量（`demo/kb_state.json`）
- 新增 `save_state()` — 将 DOCUMENTS/CHUNKS/dimension/upload_count 序列化为 JSON
- 新增 `load_state() -> dict | None` — 从 JSON 恢复 DOCUMENTS/CHUNKS
- 新增 `clear_state()` — 删除 JSON 文件
- 新增 `reset()` — 清空 DOCUMENTS/CHUNKS（cleanup 时调用）
- 在 `build_ontology()` 末尾和 `add_documents()` 末尾调用 `save_state()`

**`web_app.py`**：
- 新增 `@app.on_event("startup")` 处理器：
  1. 调用 `agent.load_state()` 加载持久化本体
  2. 检查 zvec 集合是否存在（`GET /collections/{name}`）
  3. 集合存在 → 设置 `_state["initialized"]=True`，恢复 dimension/doc_count/upload_count
  4. 集合不存在 → 调用 `agent.clear_state()`，initialized=False
- `init_kb()` 成功后调用 `agent.save_state()`
- `api_cleanup()` 增加 `agent.reset()` + `agent.clear_state()` + 清空 uploads 目录
- `upload_documents()` 成功后调用 `agent.save_state()`

### 需求 2：文档查看器 — 点击查看原文 + Markdown 渲染

**`web_app.py`**：
- 修复 `GET /api/documents` — 增加 `preview` 字段（前 100 字符），修复文档列表预览空白问题
- 新增 `GET /api/documents/{document_id}` — 返回完整文档（content + chunks + source_file + category）

**`static/index.html`**：
- `<head>` 增加 marked.js CDN（Markdown 渲染）
- 新增文档查看器模态框 HTML（全屏覆盖层）：
  - 头部：标题、来源文件、分类、关闭按钮
  - 正文：Markdown 渲染的内容 + 分块列表
  - 如有原始文件（图片/PDF/DOCX）：显示查看/下载链接
- 新增 CSS：`.modal-overlay`, `.modal-content`, `.modal-header`, `.modal-body`, `.chunk-highlight`
- `renderDocuments()` 中文档项可点击 → `viewDocument(doc_id)`
- 新增 `viewDocument(doc_id, highlight_chunk_id)` JS 函数：
  - 调用 `GET /api/documents/{id}` 获取完整文档
  - 用 marked.js 渲染 Markdown 内容
  - 展示分块列表，高亮匹配的分块
  - 如有原始文件，显示查看链接

### 需求 3：检索结果展示原文位置 + 可查看原始文档

**`web_app.py`**：
- 新增 `UPLOADS_DIR` 目录（`demo/uploads/`）
- `upload_documents()` 中保存原始文件到 `uploads/{doc_id}_{filename}`
- 文档元数据增加 `source_path` 字段
- 新增 `GET /api/files/{document_id}` — 返回原始文件（图片/PDF/DOCX）

**`static/index.html`**：
- `renderResults()` 更新：
  - 显示位置信息：heading（章节/页码）、document_id
  - 每条结果增加「查看原文」按钮
  - 点击结果或按钮 → `viewResult(doc)` 打开文档查看器
- 新增 `viewResult(doc)` JS 函数：
  - 调用 `viewDocument(doc.document_id, doc.id)`（doc.id = chunk_id）
  - 在查看器中高亮匹配的分块并滚动到该位置
  - 图片类文档：显示原始图片
  - PDF/DOCX：显示查看/下载链接 + OCR/解析的文本内容

### 不修改的文件
- `kb_data.py` — 无需修改（持久化由 agent + web_app 层处理）
- `document_loader.py` — 无需修改
- `rag_demo.py` — CLI 验证脚本不变

### 验证计划
1. 启动 zvec + web_app，初始化知识库，上传文档
2. 重启 web_app → 验证知识库自动恢复（不需要重新初始化）
3. 点击知识库文档 → 验证查看器显示原文 + Markdown 渲染
4. 搜索 → 点击检索结果 → 验证显示原文位置 + 高亮匹配分块
5. 上传图片/PDF → 搜索 → 点击结果 → 验证可查看原始文件
