## 1. 校准口径

原分析判定 Step5–10 为 🔴 缺，依据是「问答页只有输入框」。但当前 `static/index.html` 的 `doAsk()` → `askExtrasHtml()` → `renderReasoningChain(d)` 已渲染：

- **Step5** `concept_location.located_concepts`（含置信度）+ `implicit_concepts` + `constraints` + `expected_output`
- **Step6** `ontology_path[检索计划].strategies` chips
- **Step7** `ontology_path[检索执行].rounds / docs_retrieved`
- **Step8** `ontology_path[推理路径].edges`（关系边链）+ `evidence_docs`（可点击证据）
- **Step9** `trace` 按轮次分组的时间线 + `iterations`
- **Step10** `confidence` 徽章 + `uncertainties` 不确定点

> 因此"全红"不成立。但原分析点名的若干**验收级细节**确有真实缺口，本次修复。

## 3. 本次修复清单（已完成）

### 后端 `llm.py`（小改，启用前端渲染）
1. **`termination_reason` 追踪**：循环前初始化为 `"max_iterations"`，在 7 个退出点分别赋值——`timeout` / `token_budget` / `max_search_rounds` / `no_new_info` / `evidence_sufficient` / `answer_generated` / `max_iterations`。两个 return 语句均透出该字段。
2. **`_rerank` trace 透传四维分**：`top_docs` 从 `[(title, final_score)]` 改为完整 dict（含 `semantic_score / concept_score / structural_score / freshness_score / final_score / concept_ids / sources`）。reranker 早已算好，只是 trace 丢弃，现补回。
3. **`citations` 构建**：成功路径从 `all_retrieved_docs` 去重构建 `[{doc_id, title, sources, excerpt, score}]`（取前 8），两个 return 均透出。

### 前端 `static/index.html`（纯前端）
4. **快速/完整闭环徽章**：`askExtrasHtml` 渲染 `path_type` → `.path-badge.fast/deep`（绿/蓝）。
5. **反问候选可点击**（风险2 落地）：`.cand` 加 `onclick="askClarify(c)"`，新函数 `askClarify` 追加"（关于 X）"并自动 `doAsk()`。
6. **Step8 四维分条**：`renderReasoningChain` 从 `_rerank` trace 取 `top_docs`，渲染 `.rerank-scores`（4 列进度条：语义紫/概念距离蓝/结构绿/新鲜度琥珀 + 综合分）。
7. **Step9 终止原因**：Step9 段尾加 `.term-reason`，`TERM_LABELS` 映射 7 种原因为中文。
8. **Step10 结构化引用**：`askExtrasHtml` 末尾渲染 `.citations`（标题可点击→`askAboutDoc`、来源路 chip、摘要），数量标注。
