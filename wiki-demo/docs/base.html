03

命名约定 · 未覆盖控件 · 诚实说明

规范的可信度来自边界清晰：哪些已映射、哪些当前界面无图标、哪些是示意而非实物。

命名与文件约定

  • 线性操作/导航组 → 文件前缀 icon_（如 icon_search.svg）。
  • 彩色节点类型组 → 文件前缀 node_（如 node_entity.svg）。
  • 彩色状态组 → 文件前缀 state_（如 state_approved.svg）。
  • 统一 viewBox="0 0 24 24"；线性用 stroke=currentColor 以便随主题换色，彩色用固定语义 fill。
  • 两套参考表编号各自从 1 起且互相重叠，禁止用裸编号引用，一律用 key / 名称。

当前界面无专用图标的控件

  • 文档库：卡片右上角 × 移除、📊 元数据 按钮（现为内联小图，集内无专名）、正则 grep 复选框。
  • 本体：统计徽章（概念/关系/绑定/类型）、类型/关系下拉；本体图类型图例（algorithm / component / concept / configuration / infrastructure / mechanism / metric / pattern / problem / process / tool）现用色点，图标集未覆盖。
  • 知识图谱：布局切换（力导向/层次/环形/类型聚集）、图查询、所有边 下拉、边标签/箭头 开关。
  • 分析：各 W4.x 的输入框、old_ref/new_ref、定时数字输入。
  • 建议：若需图标化，优先补 11 个本体类型 glyph，使「本体类型」与「节点类型」两套图例在视觉语言上对齐。

关于本规范的三点诚实声明

  • 示意字形：本页内嵌 SVG 仅为规范内的视觉锚点，可能与仓库实际 /icons/*.svg 笔触不同；请以仓库文件为生产真源，key 与名称一一对应。
  • 文字 Tab：当前五个主 Tab 为纯文字，线性 1–5 标注为「建议图标化位置」（Tab 前缀 / 折叠侧栏 / 移动端底栏），并非现状截图已含图标。
  • 静态截图推断：「使用位置」依据所给界面截图判定；若某控件后端已实现而前端未暴露，本表标为「建议落点」。
