# 前端工作流修复计划（对齐 AI推理引擎.md）

## 设计基线
设计把系统分为**离线建图（Step1-4）**与**在线导航（Step5-10 闭环）**两条主线，核心隐喻：
> 本体 = 地图；问答 = 沿地图路径导航。地图不需完美但核心路径要正确；导航要知道何时转弯、何时停下。

在线阶段 Step5-10 应完整可见：意图理解+概念定位 → 本体展开生成检索计划 → 多策略并行检索 → Re-Rank+概念距离过滤 → 闭环评估 → 合成答案+引用+置信度+不确定。

## 当前前端问题诊断（逐项对照设计）

### 问题1：入库流程断裂（Step1-4 不可达）— P0
- **文件类型拒绝 bug**：上传区 `accept` 宣称支持 `.pdf/.html/.py/.js/.ts/.json/.yaml`，但 `handleFileSelect`/`handleFolderSelect` 只放行 `.md/.txt/.markdown`，其余**静默丢弃**，用户选 PDF 无任何提示。
- **两套入库路径未接通**：后端有 `/api/compile`（LLM 编译 raw→wiki，文本用）与 `/api/pipeline/ingest-file`（自动切分+多索引，支持 PDF/代码/HTML），前端只用 compile，ingest-file **零调用**。导致设计的 Step1 多格式解析、Step4 代码索引在前端完全不可达——"代码检索"面板存在却无法导入代码。

### 问题2：标签栏不反映两阶段心智模型 — P1
- 6 个 tab 按**数据类型**而非**工作流阶段**组织：文档库/知识图谱/元数据/本体/问答/分析。
- 新用户无法感知设计意图的顺序：先建图（导入→本体）再导航（问答）。
- "元数据"（主题/标签/实体/原始资料）是文档库的子切面，却占一个顶级 tab，信息密度低。
- "问答"（核心价值、设计主线）只是 6 个并列 tab 之一，权重被稀释。

### 问题3：在线推理链 Step5-9 严重欠可视化 — P1
后端 `/api/ask` 实际返回丰富数据，前端只渲染了一小部分：
| 设计步骤 | 后端字段 | 前端是否渲染 |
|---|---|---|
| Step5 概念定位 | `concept_location.{located_concepts,implicit_concepts,constraints,expected_output}` | ❌ 完全缺失 |
| Step6 检索计划 | `ontology_path[检索计划].strategies` | ❌ 缺失 |
| Step7 检索执行 | `ontology_path[检索执行].{rounds,docs_retrieved}` | ❌ 缺失 |
| Step8 推理路径 | `ontology_path[推理路径].{edges,evidence_docs}` | ⚠️ 边渲染了，证据文档不可点击 |
| Step9 闭环 | `iterations` + `trace` | ⚠️ 仅一个数字+折叠 JSON，无迭代时间线 |
| Step10 答案 | `answer,confidence,uncertainties,clarification` | ✅ 已渲染 |

设计的"方向感"（概念定位）和"止损能力"（闭环迭代）——两个核心价值——在前端不可见。

### 问题4：跨视图导航缺失（地图-导航隐喻断裂）— P2
- 本体概念详情 → "以此概念提问"：❌ 缺失（地图不能直接导航）
- 文档详情 → "就这篇文档提问"：❌ 缺失
- 搜索结果 → "就此结果提问"：❌ 缺失
- 答案证据文档 → 跳转文档详情：❌ 不可点击
- 问答视图与本体视图完全断连，违背设计核心隐喻。

### 问题5：工具栏冗余 + 布局问题 — P3
- 本体工具栏 9 按钮一排（新建/导出/导入/构建/修复/刷新/待审核/快照/回滚），混 CRUD/构建/版本/维护，小屏 wrapping 混乱。
- 快照/回滚在"本体"工具栏与"分析"卡片**重复出现**。
- 搜索栏的"⌘ 代码结构检索"文本按钮夹在 checkbox 中间，视觉突兀。

### 问题6：无空状态引导 — P3
- 空库只显示"暂无文档"，不引导设计的三步流程（导入→建本体→提问）。

## 修复实施（按优先级）

### Fix-1（P0）修复入库流程
- `handleFileSelect`/`handleFolderSelect`：放行全部 advertised 类型，按扩展名分类路由。
- 新增"导入文件"模式调用 `/api/pipeline/ingest-file`（多格式自动切分+索引），与"LLM 编译"（raw→wiki 文本）并列区分。
- 上传区两模式清晰标注。

### Fix-2（P1）重构标签栏
- 重排为：`文档库 → 本体 → 知识图谱 → 问答 → 分析`，元数据折叠为文档库内 toggle。
- 问答 tab 提到导航核心位置。

### Fix-3（P1）完整可视化 Step5-9 推理链
- 渲染 `concept_location`：定位概念（带置信度）、隐含概念、约束、期望输出。
- 渲染 `ontology_path` 全部 step：概念定位/检索计划(strategies)/推理路径(边+可点击证据文档)/检索执行(rounds,docs_retrieved)。
- 闭环迭代时间线：每轮 trace 分组，标注第N轮（广撒网/精确补充/验证）。
- trace 可读化：工具名映射设计步骤标签。

### Fix-4（P2）跨视图导航
- 概念详情："+ 以此概念提问"→ 切问答 tab，预填 scoped 问题。
- 文档详情："+ 就此文档提问"。
- 搜索结果：每条加"提问"按钮。
- 答案证据文档 chip 可点击 → showDocDetail。

### Fix-5（P3）工具栏清理 + 布局
- 本体工具栏分组：[新建] | [LLM构建, 修复绑定] | [快照, 回滚, 待审核] | [导出, 导入, 刷新]。
- 分析页移除重复的快照/回滚（保留本体页为版本管理主入口）。
- 代码结构检索按钮改为图标按钮。

### Fix-6（P3）空状态引导
- 空库引导卡：1.导入文档 → 2.构建本体 → 3.提问。
- 空本体引导：LLM 构建 / 导入 YAML。
