### 关键实现细节

- **W3.1 零风险兼容**：不改动 zvec 集合 schema（避免重建清空向量），而是在 `search_documents` 内对返回的 chunk 文本重新做确定性概念匹配（60s 缓存 concept 表）。若 zvec 已返回 `concept_ids`（未来 schema 含该字段）则优先使用。
- **W4.1 时间戳来源**：`_resolve_commit_ts` 用 `git show -s --format=%ct <hash>`；`_filter_since_commit` 先解析 since_commit 的 cutoff，再对候选文档按 `commit_ts`（或回退 commit_hash）做 `>= cutoff` 过滤，并保留恰好命中该 commit 的边界。
- **W4.2 与既有能力衔接**：新概念一律 `proposed`，不污染线上检索（W4.5 保证检索只用 approved）；盲区分析复用 `query_log_analysis`；快照复用 `snapshot_concepts_versions`；前端维护卡片含「立即运行 / 启动定时 / 停止」与状态展示，运行后自动刷新本体列表与待审核计数。
