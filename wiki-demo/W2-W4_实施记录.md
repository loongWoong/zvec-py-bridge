|---|---|---|---|

## Wave 3：离线索引补全（chunker.py / pipeline.py / wiki_runtime.py）

| # | 功能 | 落地位置 | 说明 |
|---|---|---|---|
| 3.1 | chunk 级概念标注 | `pipeline._annotate_chunk_concepts()` 对每个 chunk 做确定性关键词概念匹配，写入 `chunk.metadata['concept_ids']`；`chunks_to_zvec` 字段携带 `concept_ids`/`chunk_type`/`parent_summary` | chunk payload 含 concept_ids（检索端使用为后续可选项） |
| 3.2 | PDF 解析 | `pipeline.ingest_file` 增加 `.pdf` 分支（pdfplumber 懒加载，未装则明确报错）；解析为文本后按标题层级切分 | `.pdf` 加入 SUPPORTED_EXTS |
| 3.3 | HTML 解析 | `chunker.detect_type` 增加 `.html/.htm`；新增 `_strip_html`/`_split_html`（纯正则去标签取正文，chunk_type=html） | `.html/.htm` 加入 SUPPORTED_EXTS |
| 3.4 | MD 表格识别 | `chunker._is_markdown_table` + `_smart_split_paragraphs` 表格感知；表格块始终独立成 chunk，`chunk_type="table"` | 表格不被切断、可独立命中 |
| 3.5 | 超长 chunk 上下文摘要 | `_split_section_by_paragraphs` 拆出的子 chunk 写入 `metadata.parent_summary`（父块前 100 字） | 保上下文 |
| 3.6 | AST → 本体草案 | `pipeline.propose_ontology_from_directory()`：遍历代码目录用 `code_analyzer` 提取函数/类（=概念）与调用（=depends 边），生成 `ontology_draft.yaml`（需人工 review 合入，不污染线上本体） | 一键生成本体草案 |
| 3.7 | 跨库 grep 策略 | `wiki_runtime.grep_documents()` 对**全部**文档 content 正则扫描（不限绑定文件）；纳入 `_execute_search_plan` 作为独立一路 | 策略2 从「仅绑定文件」升级为独立跨库路 |
